# AI 免费额度与全站预算 V2 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use `subagent-driven-development` to execute this plan task by task, then use `verification-before-completion` before handoff.

**目标：** 普通用户按账号与设备归并后的风险主体终身获得 10,000 AI Token；平台每天最多承担 1,000,000 Token；任一额度耗尽时向用户展示后台配置的 BYOK 引导。

**架构：** PostgreSQL 保存风险主体、成员关系、终身钱包、全站日预算、预占记录和审计日志。平台调用在发起前同时预占个人额度与全站预算，完成后按保守估算用量结算，失败时释放预占。运行时策略使用 15 秒进程内缓存并支持保存后主动失效。已有 `ai_usage_events` 继续作为请求明细，BYOK 调用只记审计数据，不消耗平台额度。

**技术栈：** FastAPI、SQLAlchemy 2、PostgreSQL、Alembic、Next.js、TypeScript、pnpm workspace。

## 业务决策

- 普通用户默认终身赠送 10,000 Token，后台可修改系统默认值，也可调整指定用户所属风险主体的钱包。
- 管理员和企业用户默认不受个人免费额度约束，全站预算仍然生效。单用户钱包管理面向普通用户及其风险主体。
- 同一账号在不同浏览器登录时映射到同一风险主体；同一浏览器切换账号时，新账号会加入当前设备已绑定的风险主体。
- 设备标识由前端生成随机值并通过 `X-GEOrank-Device-ID` 发送，服务端只保存 SHA-256 摘要。清理浏览器数据、换设备并创建全新身份仍需要手机号验证、验证码或外部风控增强识别。
- 平台每天按 `Asia/Shanghai` 自然日统计，默认阈值为 1,000,000 Token。达到阈值后，所有平台模型调用进入 BYOK 引导状态。
- BYOK 不占个人钱包与全站预算。浏览器本地 API Key 继续保存在当前设备，服务端不持久化明文；退出登录和切换账号会清除旧配置。
- 后台可配置 DeepSeek 引导标题、说明、按钮文字、官网地址、默认 Base URL 和模型；用户请求的 Base URL 必须与后台允许供应商同源。
- 历史成功平台调用按用户汇总后写入首次生成的钱包，最多扣到当前赠送额度，避免升级后重复赠送。
- 预占覆盖可能的最大输出，并在成功后按实际字符估算与功能最低计费值中的较高者结算。供应商返回真实 usage 后可继续升级为真实 Token 结算。

## 已落地管理入口

- 后台「系统设置 → API 用量策略」：默认单用户额度、全站每日阈值、全站开关、应急 BYOK-only、匿名开关、BYOK 开关、供应商白名单和引导文案。
- 后台「用户管理 → AI 额度」：查询风险主体、关联账号/设备、赠送、已用、预占、剩余和冻结状态；调整时必须填写原因。
- 后台「LLM API Provider」：管理平台自身的模型供应商、Base URL、模型与密钥；密钥沿用加密存储和掩码回显。
- 前台「个人中心 → 我的模型 API」：展示个人和全站额度，额度不可用时展示后台文案及官网入口；后台关闭 BYOK 后自动停用本地配置。
- 管理接口：`GET/PUT /api/admin/api-policy`、`GET/PUT /api/admin/users/{id}/ai-quota`、`GET /api/usage/me`、`GET /api/usage/policy`。

## 当前计费与预占口径

| 功能 | 请求前预占 | 成功最低计费 | 失败处理 |
| --- | ---: | ---: | --- |
| AI 问答 | 6,500 | 1,200 | 释放 |
| 拓词 | 6,500 | 2,500 | 释放 |
| 诊断 | 10,000 | 2,500 | 最终失败释放 |
| 公司入库 | 10,000 | 6,500 | 最终失败释放 |
| AI 工具 | 4,500 | 1,200 | 释放 |

预占用于并发风控，成功后释放未结算部分。实际估算超过预占时按完整估算值入账，个人与全站账本保持一致。平台降级提示按失败事件记录，不扣个人或全站额度。

## 风控边界与下一阶段

当前设备 ID 能覆盖两类常见场景：同一账号更换浏览器仍沿用账号主体；同一浏览器切换账号会归并到设备主体。纯 Web 环境无法可靠识别清理浏览器数据后的物理设备，新浏览器与新账号组合仍可能获得新主体。

浏览器 BYOK Key 当前以明文保存在 `localStorage`，服务端不会持久化。站点发生 XSS 时本地 Key 仍有泄露风险，因此生产环境需要保持严格 CSP、依赖审计和输入输出转义。后台平台 Provider 允许管理员填写自托管 Base URL，具备访问内网地址的能力，仅应授予受信管理员并纳入后台登录保护与审计。

终端失败会关闭账本预占并记失败事件。异步流水线按已完成供应商阶段累计消耗，流式请求取消按完整预占保守扣费。现有客户端仍以字符估算为主，供应商失败切换和单次在途请求的精确 Token 需要接入真实 usage 字段；该项属于上线后的最高优先级计量升级。

生产环境第二阶段建议按优先级推进：

1. 赠送额度领取前强制完成手机号验证，并按已验证手机号建立不可重复的领取主体。
2. 注册、登录和首次 AI 调用接入 Turnstile 或同类人机验证，增加 IP/ASN/时间窗口速率限制。
3. 记录设备、账号、手机号、IP 风险关系，只保存风控所需摘要并设置留存周期。
4. 对短时间多账号、共享设备大规模切换、数据中心代理和异常高并发触发冻结或人工复核。
5. 接入供应商真实 usage 字段，替换字符估算；在完成前持续监控预占与结算差额。
6. 为 70%、85%、95%、100% 预算水位配置自动告警，并演练应急 BYOK-only 开关。

## 上线步骤

1. 先部署数据库迁移和兼容旧策略的后端；迁移默认开启 `global_budget_enabled=true`，阈值为 1,000,000。
2. 写入 V2 默认策略，确认历史用量回填和后台数据可见。
3. 部署后台与前台，验证 BYOK 引导和设备标识上报。
4. 升级后立即在后台复核全站阈值、终身赠额与 BYOK 引导；观察个人额度、预占失败率、结算差额和异常主体合并。
5. 确认平台调用全部产生预占后开启全站每日预算，先使用较高阈值观察一个自然日，再调整到 1,000,000。
6. 告警覆盖当日预算 70%、85%、95%、100%，并记录应急 BYOK-only 切换审计。

## 回滚原则

- 应用回滚前先切换到 BYOK-only 并停止新异步任务，保留账本数据，旧版继续读取原 `ai_usage_events` 和 `user_daily_usage`。
- 数据库 downgrade 只在确认没有 V2 应用实例后执行。
- downgrade 会把旧策略切到旧版可识别的 `byok_required`，避免旧版把新模式解释为无限平台额度。
- 预占记录超过超时时间自动释放，恢复平台调用前先运行过期预占回收。
- 管理员额度调整均有审计，可通过反向调整恢复，不直接删除账本。
