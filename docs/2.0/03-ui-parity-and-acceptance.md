# 03. UI 保真与验收标准

## 核心要求

2.0 迁移过程中，不允许把重构变成重设计。

必须保留：

- 模块顺序
- 信息层级
- 交互路径
- 页面语气和视觉气质

允许优化：

- 响应式稳定性
- 粘性侧栏、布局规则
- 组件一致性
- 可维护性
- 可访问性

## 页面验收维度

每个页面迁移后按以下 5 项验收：

1. 视觉对齐
- 主要版式不变
- 颜色体系不漂移
- 按钮与卡片气质一致

2. 信息对齐
- 模块没有缺失
- 模块顺序不乱
- 关键信息字段完整

3. 交互对齐
- 原页面关键动作仍然成立
- URL 规则兼容
- 登录和权限行为一致

4. SEO/GEO 对齐
- title
- description
- canonical
- schema
- HTML 直出正文

5. 业务对齐
- 对应接口返回正确
- 关键流程可走通

## 页面分组

### 第一批必须保真

- 首页
- 教程频道
- 教程详情
- 公司详情

### 第二批必须保真

- 诊断工具页
- 诊断报告详情
- 方案对话页
- 拓词页

### 第三批必须保真

- 后台仪表盘
- 公司管理
- 诊断管理
- 方案管理
- 拓词管理
- 教程管理

## 默认验收方式

- 页面截图比对
- 真实点击路径比对
- HTML 源码检查
- SEO 字段检查
