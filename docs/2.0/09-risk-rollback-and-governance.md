# 09. 风险、回滚与治理

## 主要风险

### 1. 旧 URL 权重丢失

风险：

- 公开内容页迁移后，如果路径变动不兼容，会影响 SEO/GEO 索引。

处理：

- 旧路径 301
- 新旧 canonical 明确
- 保留旧实体映射表

### 2. UI 漂移

风险：

- 重构过程中把保真迁移做成重新设计。

处理：

- 逐页截图对照
- 每页必须通过 UI 保真清单

### 3. 认证切换风险

风险：

- localStorage token 到新 session 体系切换时，前后台行为不一致。

处理：

- 保留兼容期
- 新旧 token 逻辑短期并存
- 分模块切换

### 4. 多语言返工风险

风险：

- 如果先上页面翻译，后补数据模型，会导致重复改路由和 SEO。

处理：

- 先设计 translation 表
- 先设计 locale slug

## 回滚规则

### 页面级回滚

- 新页面上线后发现重大回归，可切回旧页面入口

### 模块级回滚

- 单个模块切换失败，仅回滚该模块，不影响其他模块

### 数据级回滚

- Alembic 迁移必须可逆或具备补偿脚本

## 治理规则

- 新页面必须通过 UI 验收
- 新 API 接口必须进入 OpenAPI
- 新前端调用必须走 SDK
- 新后台页面必须走共享组件
- 新多语言字段必须进入 translation 模型
